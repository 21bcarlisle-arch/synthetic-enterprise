The Director's Window

The authenticated ops pane. It does two jobs and no more this pass: the reserved queue — the decisions the machine has parked for your own hand and cannot take itself — and machine health — the live daemon set, honest about what is stale. Every figure is generated from repo data only. Delta-view, the decision battery and the stop control are deliberately deferred (Spec-005 §3); an ops pane that cannot act is honest, one that fakes a write control would not be.

How this pane is gated. It is off-nav and noindex — unlisted, reachable only by knowing the URL. The view itself is not behind a lock: it renders read-only repo state, and a client-side PIN that opened with any key would be theatre, not a control (R15). The authenticated affordance is the comment box (bottom-left): it reuses the site's auth pattern — a PIN checked server-side by background/director_comments.py before anything is ever staged, the only place a browser PIN means anything here. Real director writes (repo/keys/curriculum/fitness) stay console-only (one-way-door categories 5/6/8); the decision battery and stop control that would need genuine write-auth are deferred this pass, so there is nothing to fake-gate.
