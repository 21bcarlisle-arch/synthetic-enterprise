## Binding rules from the brief (enforce site-wide, from the first migrated page)
1. Every claim links to its evidence (data file + commit / map cell / test).
2. Every number carries a passport: basis (billed/settled/banked), freshness
   stamp, PROVISIONAL badge where M4 applies.
3. The site is a rendering, never an author — generated from repo data only.
4. Honesty featured: simplifications, open defects, adjudications are visitor
   surfaces, not buried artefacts.
5. Progressive disclosure: headline → mechanism → artefact → code.
6. **(v4) Claim-status on every published figure.** Each figure carries its status from the pitch's
   Note on Claims: **observed-with-evidence · external-benchmark · chosen-not-derived ·
   not-yet-instrumented · hypothesis-with-a-designed-test · retracted**. This extends rule 2's
   passport (basis/freshness/provisional) — a figure whose status is ambiguous is a defect. The
   pitch stakes a public claim that the company "publishes what it decided and why, and corrects
   itself in public"; the site is where that is true or false.
7. **(v4) Time-sensitive honesty facts must stay accurate** (load-bearing, will expire): *no
   external person has yet seen this work · no household data has entered the system · no volunteer
   has been approached.* Any surface asserting these must be data-driven so it cannot silently drift
   false once they change.
