- **Horizons, epochs, value-cycle stages, the function matrix** are all **family ŌæĀ** ŌĆö they are *the same atom set grouped by a different column*. ### 1.2 The test: one answer to "what / in what order / at what level"

Given the framework, the three questions collapse to **one function evaluated over one table**:

- **What to build?** = the **output of the draw** (`_maturity_map_draw_concurrent`) ŌĆö the atom(s) that survive the filters, weighted by dial.
- **In what order?** = the draw's **precedence**: authorization (front/gate) ŌåÆ eligibility (loop-stage) ŌåÆ dial weight ŌåÆ tie-break (compounding-return, then critical path). Deterministic given the atom fields + the director's dials/fronts.
- **At what level?** = the atom's **loop_stage** advancing `level_current ŌåÆ level_target`, each step earned by passing the loop and (for L3) the coupled gap.

There is no second answer because there is no second object. The epoch arc doesn't answer "what next" ŌĆö it *describes* the campaign; the draw answers it. The value cycle doesn't answer "at what level" ŌĆö it *groups* for the veteran's eye; the loop answers it. **Every framing is a lens on the atom; the draw is the only decider.**
