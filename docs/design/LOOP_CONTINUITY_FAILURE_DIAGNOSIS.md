# Diagnosis — "running continuous" broke (09:12→10:44 idle; origin-staged doc unconsumed)

**Provenance:** director, 2026-07-19: *"checkpointed at 09:11 saying 'running continuous' then idled 93 minutes. An advisor doc unconsumed in docs/staging/ since 10:32 didn't wake you… second occurrence today."* Evidence-based (R9), not reconstruction.

## What actually happened (evidence)
- `docs/observability/pull-loop-log.md` last entry: **`[09:12:04 UTC] drained-and-gated -> allow stop (quiet wait)`**. No entries after. The loop went dormant at 09:12 and never re-drew.
- `find_work(False)` called live now returns **`(None, False)`** — the treadmill-quiet third state — with its own log: `THREE-LANE self-refill: BUILD=0, SITE=0, DISCOVERY=0` · `all 30 idle atoms are FRAME-saturated — returning []` · `DRAINED-AND-GATED quiet wait … a genuinely new signal wakes it`.
- The advisor doc `DIRECTOR_AMENDMENT_TOKEN_RESOURCE_DIMENSION_2026-07-19.md` was committed to **origin** at 10:32 (`1c3baea28`, `[ADVISOR-STAGED]`) but was **never in the local working tree** `find_work` reads. Separately, the local `process_run_complete` daemon had made **7 auto-process commits (10:19–11:51) never pushed** — the local↔origin trees had drifted in *both* directions.

## Three stacked root causes (all necessary for the observed failure)

**RC1 — the loop went quiet because there was no mechanically-drawable work.**
All 30 idle atoms (including the gate-after-authorized Epoch-2 campaign A–G) sit at `loop_stage: idle`, which the draw treats as BUILD-gated (offers them for DISCOVER/FRAME only, never BUILD). Their DISCOVER/FRAME is saturated (done). So the draw correctly found zero below-target work and rested. **The gate-after directive authorized campaign BUILD as POLICY but it was never MECHANIZED into the draw** — the atoms were never flipped to a drawable stage and there's no `FRONT_OPEN` in `gate_authorizations.jsonl` for the campaign. I compounded this by building campaign work via *manual fork dispatch* rather than through the draw — so the moment I stopped hand-dispatching, `find_work` had nothing and the treadmill-quiet mechanism (correctly, given the map state) rested.

**RC2 — RETRACTED after checking the evidence (R9). The re-arm EXISTS and WORKED.**
My first pass asserted "a dormant loop has no re-arm." **That was wrong — an inferred narrative I had not checked.** `docs/observability/supervisor-log.md` proves the `supervisor` daemon (systemd, sole turn-granting authority) cycled every ~2 minutes throughout the idle window (10:46, 10:48, 10:50, 10:52 …), calling `find_work` each cycle and — correctly — not granting a turn because `find_work` returned "drained-and-gated, no work." It is NOT dormant and NOT inert; it faithfully re-checks and would poke the idle seat the instant `find_work` returns work. So the re-arm is healthy; the failure is upstream of it — `find_work` genuinely had nothing to hand it (RC1), and the one new piece of work (RC3) was invisible to the local tree it reads. **Lesson (self-caught): I nearly shipped a wrong root cause; the supervisor log settled it in one read.**

**RC3 — even a live loop would not have seen the doc.**
Advisor docs arrive via the staging bridge as commits to **origin**. The local working tree does not auto-pull, and `find_work` reads the LOCAL `docs/staging/`. So an origin-committed advisor doc is invisible to the local loop until someone pulls (the same sync-latency that hid the Epoch-2 campaign earlier this session). The 7 unpushed local daemon commits are the mirror image.

"Running continuous" was still an overclaim — it holds only while `find_work` keeps returning work, and I had drained the local drawable work without wiring the authorized campaign BUILD into the draw, while the one new directive sat on origin.

## Fix proposal (RC1 + RC3 are the real remedies; RC2 needs nothing — the supervisor already works)
- **RC1 → give `find_work` real work.** Two parts: (a) the amendment's **OPTIONAL lane** — forward-discovery the loop draws when idle + headroom, so it rarely starves ("something valuable to reach for instead of busywork or nothing"); (b) **mechanize the authorized campaign BUILD into the draw** — flip the A–G atoms to a drawable stage so the supervisor can hand them out. That flip needs the campaign `FRONT_OPEN` in `gate_authorizations.jsonl`, which is **director-console-only** (I must not self-write authorization) — so this is flagged for the director's console act. Until then, campaign BUILD proceeds by explicit dispatch (as now), which is why the loop starves when I stop dispatching.
- **RC3 → the local draw must see origin (the sharp, narrow fix).** The `supervisor`/`find_work` work-check does NOT `git fetch`; `staging_watcher` fetches + notifies Rich but does not land origin docs into the local working tree or wake the seat. So an origin-`[ADVISOR-STAGED]` directive is invisible to the local draw until a human pulls. Fix: the work-check `git fetch`es and includes origin's `docs/staging/` (or staging_watcher checks the file out locally + the supervisor treats it as a wake). Bidirectionally, local daemon commits must push (7 sat unpushed today). This is the highest-value, most-contained fix and directly closes "the advisor doc didn't wake me." R15-failable: a doc committed to origin-only must, within one supervisor cycle, become visible to `find_work`.
- **RC2 → no change needed.** The supervisor re-arm is healthy (proven above).

**Interim mitigation in force:** the loop is kept alive this turn by active work (two forks + main-seat). I will not claim "continuous" again until RC3 is closed and campaign BUILD is drawable — because until then it depends on me hand-feeding the draw, which is exactly what failed.
