## Implementation change: two new functions in `sim/risk_engine.py`

Two of the requested risk-engine tests (`test_coc_deducted_from_net_margin`,
`test_administration_triggers_at_zero_treasury`) describe logic that existed only
as ad-hoc inline code duplicated across `simulation/run_phase*.py` scripts
(`record["margin_gbp"] - capital_cost_per_period_gbp` and
`if treasury <= 0 and administration_event is None`). To give these tests a real,
reusable target — and per the new Harness Rule ("every new feature instruction
must include the test that proves it works") — two small pure functions were added
to `sim/risk_engine.py`:

- `compute_net_margin(gross_margin_gbp, capital_cost_gbp) -> float`
- `is_administration_triggered(treasury_balance_gbp) -> bool`

These consolidate existing behaviour; the `simulation/run_phase*.py` scripts were
not refactored to call them (left as a follow-up) — they continue to work as-is.

## What the harness now covers

- **`saas/tariff_pricing.py`** — tariff always covers forward price and capital
  cost; documents that the capital-cost term is currently customer-size-independent;
  confirms the post-2023 regulatory sigma floor produces a strictly higher tariff
  than pre-2023.
- **`sim/risk_engine.py`** — VaR scales with naked volume; the post-2023 stressed
  floor (1.50) binds over a calmer current-conditions view; net margin =
  gross margin − capital cost; administration triggers at treasury ≤ 0.
- **`sim/forward_curve.py`** — forward price exceeds flat spot due to seasonal
  loading; higher `risk_factor` strictly increases the forward price; the forward
  price is point-in-time safe (future-dated records do not change the result).

## Environment setup (not part of the repo)

`pandas`, `numpy`, `pytest`, and `ruff` were not previously installed. No system
Python, sudo, or apt access was available for `python3-venv`/`make`, so:
- Packages installed via `python3 -m pip install --user --break-system-packages
  pandas numpy pytest ruff` (user site-packages, fully reversible — `pip uninstall`
  to remove).
- `make` (absent, no `apt install` permission) was obtained via
  `apt-get download make` + `dpkg -x` into `~/.local/opt/make`, symlinked into
  `~/.local/bin`.
- `~/.local/bin` must be on `PATH` to run `make check` / `ruff` / `pytest` directly.
