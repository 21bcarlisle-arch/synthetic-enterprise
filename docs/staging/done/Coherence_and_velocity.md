# Strategic Coherence Infrastructure

## The Problem

Rich and the strategy advisor have staged multiple documents over
several days — destination vision, architecture upgrade, dashboard
vision, invoice and portal, product framing. The agent actions them
as a queue of disconnected tasks without understanding how they
relate or what the current priority stack is. Neither Rich nor the
advisor can easily see what's been staged, what's been actioned,
or what's pending.

This instruction creates three things to fix that:

1. A living priority file — the authoritative work queue
2. A staging status page — visibility into what's queued and done
3. A build timeline and velocity dashboard — evidence of progress

---

## 1 — PRIORITIES.md (the authoritative work queue)

Create `docs/PRIORITIES.md`. This is a living document, updated
after every session. It is the single source of truth for what
matters most right now.

Format:

```
# Current Priorities

Last updated: [date] by [Rich/advisor/agent]

## Now (active this session)
1. [item] — [why it's first] — [status]

## Next (queued, unblocked)
2. [item] — [why it's next] — [depends on: X]
3. [item]

## Staged (in docs/staging/, awaiting action)
- [filename] — [what it contains] — [staged date]

## Backlog (agreed but not yet scheduled)
- [item from DESTINATION_VISION.md backlog]

## On hold (deprioritised, reason)
- [item] — [reason]

## Recently completed (last 5)
- [phase/item] — [date]
```

**Rules:**
- Never more than 3 items in "Now"
- Every staged document must appear in "Staged" section
- Agent updates this file at the start of every session and after
  every phase close
- Rich and advisor update the "Now" and "Next" sections when
  steering changes
- Published to poesys.net/priorities/ after every update

## 2 — Staging Status Page

Create `site/staging-status/index.html` published to
poesys.net/staging-status/

This page shows:

**Staged (awaiting action):**
Table of files in docs/staging/ with: filename, date staged,
brief description (first line of file), estimated effort

**In progress:**
Current phase and what staged instruction it comes from

**Recently actioned:**
Last 10 items from docs/staging/done/ with: filename, date actioned,
phase it became, outcome (tests added, capability built)

**Ignored / not found:**
Any staged files older than 7 days that haven't been actioned —
flagged as needing attention

This page is regenerated after every session. Rich and the advisor
can see at a glance whether staged instructions are being picked up.
