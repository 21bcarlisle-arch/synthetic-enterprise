# Proposed Phase MX -- Company Payment Behaviour Analytics

**Drafted:** 2026-07-01T23:24:33Z
**4-hour opt-out window expires:** ~2026-07-02T03:24:33Z
**Replaces:** Phase MW proposal (Phase MW now complete)

## Summary

Build the company-side PaymentBehaviourAnalytics module that scores customers based on
observable payment records generated by Phase MW. The canonical HSL scenario is now
completable end-to-end: job_loss (SIM) -> HIGH stress (SIM) -> LATE/DD_FAILED records
(observable) -> company observes behaviour score degrading -> company responds.

## Why now

Phase MW wired income_stress to synthetic payment records (LATE/DD_FAILED patterns).
But the company has no module to read those signals yet. Phase MX closes the loop:
the company builds a payment behaviour model from observables, independent of SIM
internals. This also addresses the Board Risk RED item 'Churn blind miss rate' -- 
deteriorating payment behaviour is a leading churn indicator.

## Files to create

company/crm/payment_behaviour_analytics.py -- new company module:
  BehaviourScore enum: EXCELLENT/GOOD/FAIR/POOR/CRITICAL
  score_payment_history(records: list[dict]) -> BehaviourScore
    - EXCELLENT: on_time_rate >= 0.95, no DD_FAILED
    - GOOD:      on_time_rate >= 0.80, dd_fail_rate < 0.05
    - FAIR:      on_time_rate >= 0.60, dd_fail_rate < 0.15
    - POOR:      on_time_rate >= 0.40, dd_fail_rate < 0.35
    - CRITICAL:  otherwise
  compute_payment_metrics(records: list[dict]) -> dict
    - on_time_rate, late_rate, dd_fail_rate, avg_days_late
  PaymentBehaviourAnalytics:
    record_payment(customer_id, record: dict) -> None
    get_score(customer_id) -> BehaviourScore | None  (None if no history)
    get_metrics(customer_id) -> dict | None
    is_at_risk(customer_id) -> bool  (POOR or CRITICAL)
    at_risk_customers() -> list[str]
    score_trend(customer_id, window: int) -> str  (IMPROVING/STABLE/DETERIORATING)

## Epistemic note

PaymentBehaviourAnalytics reads payment records (observable). It does NOT read
income_stress (SIM ground truth). The score is the company's model of payment
quality, built from observed outcomes. Consistent with SIM/company barrier.
